# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

Instead, please email **security@devlabs.com** with:

1. A description of the vulnerability.
2. Steps to reproduce.
3. The potential impact.
4. Any suggested fix (if applicable).

We will acknowledge receipt within **48 hours** and aim to provide a fix or mitigation within **7 business days** for critical issues.

## Scope

The following are in scope:
- Authentication bypass or API key leakage in the SDK.
- License validation bypass.
- Cache poisoning or cross-tenant data leakage.
- Dependency vulnerabilities that affect SDK consumers.

## Disclosure

We follow [coordinated disclosure](https://en.wikipedia.org/wiki/Coordinated_vulnerability_disclosure). We will credit reporters in the release notes unless they prefer to remain anonymous.
